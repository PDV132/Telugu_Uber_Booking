Features Highlighted

        Dynamic Agent Reasoning: LangChain ReAct decides which tool to use at each step.

        Hybrid Async Flow:

        Booking is simulated.

        Monitoring runs in background asynchronously.

        Consent & Privacy Enforcement: User approves before translation or ride booking.

        Notifications: Telegram & Twilio handle updates and emergencies.

        Modular & Extensible: You can swap ASR, translation, NER, or notifications without changing the core agent.

What Happens During Simulation

                Speech-to-text: Converts “voice_input.wav” → text (simulated).

                Language detection: Determines Telugu / English / Hindi.

                Translation: Telugu → English if needed.

                Intent recognition: Determines if the user wants to “book”, “cancel”, or “schedule”.

                Entity extraction: Pickup/drop, time, payment mode.

                Simulated booking: Prints ride object (no real Uber call).

                Async monitoring:

                Monitors route every few seconds.

                Prints “on-route” or “off-route” alerts.

                Notifications: Simulates Telegram and SMS messages.

Safety Features

        No real Uber API calls are made.

        All ride objects are simulated.

        Async monitoring runs in background but only prints/logs alerts.

        Consent checks are simulated.
